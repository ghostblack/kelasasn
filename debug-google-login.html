📖 Troubleshooting Guide

⚠️ Common Issues:
  • Login Loop: User ada tapi terus redirect ke login
    • Cause: Email verification check salah untuk Google users
    • Fix: Update ProtectedRoute dan AuthContext
  • Session Invalid: Session di localStorage tidak valid di Firestore
    • Cause: Session di-invalidate sebelum dibuat
    • Fix: Create session DULU, baru invalidate others
  • Race Condition: AuthContext check sebelum LoginPage selesai
    • Cause: onAuthStateChanged trigger multiple times
    • Fix: AuthContext harus selalu set user, tidak pernah null jika authenticated
✅ Expected Flow (Google Login):
  1. User click "Lanjutkan dengan Google"
  2. Redirect ke Google → Select account
  3. Google redirect back → getRedirectResult()
  4. Create/update user profile in Firestore
  5. Create new session → Get sessionId
  6. Invalidate other sessions (pass sessionId)
  7. Store session in localStorage
  8. Navigate to /dashboard or /setup-username
  9. AuthContext detect user → Set user (NO NULL!)
  10. ProtectedRoute check → isGoogleUser = true → Allow
  11. Dashboard loads successfully ✅
